## 1. AI는 어떻게 작동하는가

### 핵심 개념
AI(인공지능)는 "스스로 생각하는 컴퓨터"가 아니다. **대량의 텍스트 데이터에서 패턴을 학습한 확률 예측 시스템**이다.

### 비유
도서관에 있는 모든 책을 읽은 사람이 있다고 상상하자. 이 사람에게 "서울에서 유명한 관광지는?"이라고 물으면, 읽었던 수많은 책에서 "서울", "관광지"와 함께 자주 등장했던 단어들을 조합해서 대답한다.

AI도 비슷하다:
1. 인터넷의 수십억 페이지 텍스트를 학습 (= 책을 읽는 과정)
2. "이 단어 다음에 올 가능성이 가장 높은 단어는?" 패턴을 익힘
3. 질문을 받으면 학습된 패턴을 기반으로 가장 그럴듯한 답을 생성

### 중요한 점
- AI는 "이해"하는 게 아니라 "예측"하는 것이다
- 학습 데이터에 없는 정보는 모른다 (또는 지어낸다 = hallucination)
- 같은 질문에도 매번 다른 답이 나올 수 있다 (확률 기반이니까)

## 2. 임베딩 — AI가 언어를 이해하는 방법

### 문제
컴퓨터는 숫자만 이해한다. "사과"라는 단어를 컴퓨터에게 어떻게 전달할까?

### 임베딩이란
단어나 문장을 **숫자 벡터(숫자 목록)**으로 변환하는 기술이다.

```
"사과" → [0.23, -0.15, 0.87, 0.42, ...]   (수백~수천 개의 숫자)
"배"   → [0.21, -0.12, 0.85, 0.38, ...]   (사과와 비슷!)
"자동차" → [0.91, 0.45, -0.33, 0.12, ...]   (사과와 매우 다름)
```

### 왜 중요한가
- 의미가 비슷한 단어는 숫자도 비슷하다 → AI가 "유사성"을 판단할 수 있다
- "은행원 자격증"을 검색하면 "금융 자격증"도 관련 결과로 나오는 이유가 이것
- 이미지, 음성도 같은 원리로 숫자 벡터로 변환된다

### 일상 비유
지도에서 서울과 인천은 가깝고, 서울과 뉴욕은 멀다. 임베딩은 단어를 "의미의 지도" 위에 배치하는 것이다.

## 3. LLM(대규모 언어 모델)의 구조

### LLM이란
Large Language Model. 수십억~수조 개의 수학적 파라미터(가중치)로 이루어진 거대한 패턴 인식 시스템.

### 주요 모델들
| 모델 | 회사 | 특징 |
|------|------|------|
| Claude (Opus, Sonnet, Haiku) | Anthropic | Lulu가 사용하는 모델. 코드, 분석에 강함 |
| GPT-4o | OpenAI | ChatGPT의 엔진 |
| Gemini | Google | 구글 검색과 통합 |
| DeepSeek | DeepSeek | 오픈소스, 저렴 |

### 모델의 크기
- Haiku: 가벼움, 빠름, 저렴 — 간단한 대화
- Sonnet: 중간 — 일반 업무
- Opus: 가장 크고 똑똑함 — 복잡한 분석, 코딩 (Lulu가 사용)

크기가 클수록 더 정확하지만, 느리고 비싸다.

### 작동 과정 (단순화)
```
입력: "한국의 수도는"
  ↓
토큰화: ["한국", "의", "수도", "는"]
  ↓
임베딩: 각 토큰을 숫자 벡터로 변환
  ↓
Transformer 연산: 단어 간 관계 계산 (수십억 번의 곱셈)
  ↓
확률 계산: "서울" 95%, "부산" 2%, "대전" 1% ...
  ↓
출력: "서울"
  ↓
다시 입력에 추가: "한국의 수도는 서울"
  ↓
다음 토큰 예측: "입니다" 80%, "이다" 15% ...
```

이걸 한 토큰씩 반복해서 전체 문장을 생성한다. 이것을 **Auto-regressive generation**이라고 한다.

## 4. 토큰 — AI의 읽기 단위

### 토큰이란
AI가 텍스트를 처리하는 최소 단위. 단어보다 작을 수 있다.

```
"안녕하세요" → ["안녕", "하세요"]  (2토큰)
"Hello world" → ["Hello", " world"]  (2토큰)
"AI 에이전트" → ["AI", " 에이", "전트"]  (3토큰)
```

### 왜 중요한가
- AI 서비스 요금은 토큰 수로 계산된다
- Claude Opus: 입력 $15/백만 토큰, 출력 $75/백만 토큰
- 한국어는 영어보다 토큰을 더 많이 사용한다 (같은 의미라도 1.5~2배)

### 대략적 환산
- 영어 1,000토큰 ≈ 750단어 ≈ A4 1페이지
- 한국어 1,000토큰 ≈ 500자 ≈ A4 반 페이지

## 5. 컨텍스트 윈도우 — AI의 단기 기억

### 개념
AI가 한 번에 "볼 수 있는" 텍스트의 최대 길이. 사람의 단기 기억과 비슷하다.

```
┌─────────────────────────────────────────────────┐
│              컨텍스트 윈도우 (200K 토큰)            │
│                                                   │
│  [시스템 프롬프트] [이전 대화들...] [현재 질문]      │
│                                                   │
│  ← 이 안에 있는 것만 기억한다 →                     │
└─────────────────────────────────────────────────┘
```

### 한계
- Claude: 약 200,000 토큰 (책 1~2권 분량)
- 윈도우를 넘어가면 오래된 대화는 잊어버린다
- 이것이 "메모리"가 필요한 이유 (뒤에서 설명)

### Lulu에서의 적용
Lulu는 대화가 길어지면 자동으로 이전 내용을 요약(compact)한다. hooks/pre-compact.sh와 hooks/post-compact.sh가 이 과정에서 중요한 정보를 보존하는 역할을 한다.

## 6. 프롬프트 엔지니어링

### 프롬프트란
AI에게 보내는 지시/질문 텍스트. 같은 AI라도 프롬프트에 따라 결과가 완전히 달라진다.

### 나쁜 프롬프트 vs 좋은 프롬프트

**나쁜 예:**
```
이력서 써줘
```

**좋은 예:**
```
나는 경영학과 4학년이고, 은행 텔러 포지션에 지원하려고 해.
보유 자격증: 은행FP, 컴퓨터활용능력 1급
인턴 경험: KB국민은행 여름 인턴 (2개월)

위 정보를 기반으로 은행 텔러 지원용 자기소개서 초안을 작성해줘.
분량은 800자 내외, 성실함과 고객 응대 경험을 강조해줘.
```

### 프롬프트 작성 원칙

1. **구체적으로**: "잘 써줘" ❌ → "800자 내외, 경력 위주로" ✅
2. **역할 부여**: "너는 금융 분야 전문 컨설턴트야" → AI가 해당 관점에서 대답
3. **형식 지정**: "표 형태로", "번호 매겨서", "마크다운으로"
4. **예시 제공**: 원하는 결과물의 예시를 보여주면 품질 향상
5. **단계별 사고 요청**: "단계별로 생각해봐" → 복잡한 문제에서 정확도 향상

### 고급 기법

**Few-shot prompting** — 예시를 몇 개 주는 것:
```
자격증 정보를 아래 형식으로 정리해줘:

예시:
- 자격증명: 은행FP
- 발급기관: 한국금융연수원
- 취득일: 2025-03-15
- 유효기간: 영구

이제 아래 이미지에서 자격증 정보를 추출해서 같은 형식으로 정리해줘.
```

**Chain-of-thought** — 사고 과정을 명시적으로 요청:
```
이 관광 상품의 수익성을 분석해줘.
1. 먼저 비용 항목을 나열하고
2. 예상 매출을 계산하고
3. 손익분기점을 산출해줘
```

## 7. 시스템 프롬프트와 페르소나

### 시스템 프롬프트란
AI에게 "너는 누구이고, 어떻게 행동해야 하는지" 알려주는 설정 텍스트. 사용자의 질문보다 먼저 AI에게 전달된다.

```
┌──────────────────────────┐
│ 시스템 프롬프트            │  ← AI의 "성격"과 "규칙" 설정
│ (CLAUDE.md)              │
├──────────────────────────┤
│ 사용자 메시지             │  ← 실제 대화
│ "이력서 써줘"             │
├──────────────────────────┤
│ AI 응답                  │  ← 시스템 프롬프트의 규칙을 따라 응답
│ "..."                    │
└──────────────────────────┘
```

### Lulu에서의 적용 — CLAUDE.md

Lulu의 `CLAUDE.md` 파일이 바로 시스템 프롬프트이다:

```markdown
# Lulu — Personal AI Assistant

## Identity
You are Lulu, a personal AI assistant...

## Principles
1. Conclude first, reason second.
2. Be practical.
3. No sycophancy.
...

## Capabilities
### Document & File Processing
### Portfolio & Career Support
### Domain Knowledge
- 은행/금융: 금융 용어, 은행 업무 프로세스...
- 관광기획: 관광 산업 트렌드...
```

이 파일이 매 세션 시작 시 AI에게 전달되어, Lulu가 "은행/관광 분야에 밝고, 파일을 분석하는 개인 비서"로 행동하게 만든다.

### 페르소나 설계의 핵심
1. **정체성**: 이름, 역할, 전문 분야
2. **원칙**: 어떤 톤으로, 어떤 방식으로 대화할지
3. **능력**: 무엇을 할 수 있고 무엇을 못 하는지
4. **제한**: 위험한 행동에 대한 가드레일

## 8. 메모리 — AI에게 장기 기억 만들어주기

### 문제
AI는 대화가 끝나면 모든 것을 잊는다. 어제 "나는 경영학과 4학년이야"라고 말했어도, 오늘 새 대화에서는 기억하지 못한다.

### 해결: 파일 기반 메모리

```
~/Documents/Lulu_Memory/
├── session_notes/lulu/        # 매일의 대화 기록
│   ├── 2026-04-15.md          # 오늘 뭘 했는지
│   └── 2026-04-14.md          # 어제 뭘 했는지
├── inbox/                     # Discord에서 받은 파일들
└── checkpoint.md              # 대화 중간 저장점

.claude/memory/
├── MEMORY.md                  # 기억 목록 (인덱스)
├── user_profile.md            # "경영학과 4학년, 은행FP 보유"
├── portfolio_progress.md      # "이력서 초안 완료, Notion 50% 작성"
└── ...
```

### 작동 방식
1. 대화 중 중요한 정보 발견 → 메모리 파일에 저장
2. 다음 세션 시작 → `hooks/session-start.sh`가 메모리 파일을 읽어서 AI에게 전달
3. AI가 "아, 이 사용자는 경영학과 4학년이고 은행 지원 준비 중이구나"라고 컨텍스트 파악

### 메모리 유형
| 유형 | 예시 | 설명 |
|------|------|------|
| user | "경영학과 4학년, 은행FP 보유" | 사용자에 대한 정보 |
| feedback | "표 형태 정리를 선호함" | 사용자의 선호/피드백 |
| project | "Notion 포트폴리오 50% 완성" | 진행 중인 작업 |
| reference | "은행FP 시험 일정: 6월 15일" | 참고 정보 |

### Lulu에서의 적용
Lulu는 대화하면서 자동으로 메모리를 쌓는다. "나 토익 850점이야"라고 말하면, 다음 세션에서도 기억하고 있다.

## 9. 도구 사용(Tool Use) — AI가 행동하는 방법

### 문제
AI는 기본적으로 "텍스트만 생성"할 수 있다. 파일을 읽거나, 메시지를 보내거나, 계산을 하려면?

### 도구(Tool)란
AI가 호출할 수 있는 "기능 버튼"이다.

```
사용자: "이 PDF 요약해줘" + [파일 첨부]
  ↓
AI 생각: "PDF를 읽어야 하네 → Read 도구를 사용하자"
  ↓
AI → [Read 도구 호출: /path/to/file.pdf]
  ↓
도구 실행 → PDF 내용 반환
  ↓
AI: "이 문서는 은행 업무 매뉴얼이고, 주요 내용은..."
```

### Lulu가 사용하는 도구들

| 도구 | 기능 | 예시 |
|------|------|------|
| Read | 파일 읽기 | PDF, 이미지, 텍스트 파일 읽기 |
| Write | 파일 쓰기 | 정리된 내용을 파일로 저장 |
| Edit | 파일 수정 | 기존 문서 일부 수정 |
| Bash | 명령어 실행 | 폴더 만들기, 프로그램 실행 |
| Glob | 파일 검색 | "Downloads 폴더에서 PDF만 찾기" |
| Grep | 내용 검색 | "이 폴더에서 '자격증' 포함된 파일 찾기" |
| WebSearch | 웹 검색 | 최신 정보 검색 |
| reply | Discord 메시지 전송 | 분석 결과를 Discord로 답장 |

### 도구 사용의 흐름
```
1. 사용자가 요청
2. AI가 어떤 도구가 필요한지 판단
3. 도구 호출 (파라미터 포함)
4. 도구 실행 결과 수신
5. 결과를 바탕으로 답변 생성
6. 필요하면 다른 도구 추가 호출
```

## 10. MCP(Model Context Protocol)

### MCP란
AI가 외부 서비스와 소통하는 **표준 규격**. USB가 다양한 기기를 컴퓨터에 연결하는 표준인 것처럼, MCP는 다양한 서비스를 AI에 연결하는 표준이다.

### 비유
```
USB 없던 시절:                     USB 등장 후:
프린터 → 프린터 전용 케이블        프린터 → USB
키보드 → 키보드 전용 케이블        키보드 → USB  
마우스 → 마우스 전용 케이블        마우스 → USB

MCP 없던 시절:                     MCP 등장 후:
Slack → Slack 전용 코드            Slack → MCP
Discord → Discord 전용 코드        Discord → MCP
Notion → Notion 전용 코드          Notion → MCP
```

### MCP 서버와 클라이언트
```
┌──────────────┐     MCP 프로토콜     ┌──────────────────┐
│  AI (Claude)  │ ←──────────────── │  MCP 서버         │
│  = 클라이언트  │ ────────────────→ │  (Discord, Notion) │
└──────────────┘                    └──────────────────┘
     "reply 도구로                      Discord에 메시지
      메시지 보내줘"                     실제 전송
```

### Lulu에서의 MCP 구조

```
┌──────────────────────────────────────────┐
│                Lulu (Claude Code)          │
│                                           │
│  도구 목록:                                │
│  - reply (Discord에 답장)                  │
│  - send_file_to_discord (파일 전송)         │
│  - dismiss (반응 제거)                      │
│  + Read, Write, Edit, Bash... (내장 도구)   │
└────────────┬─────────────┬───────────────┘
             │             │
    ┌────────▼────────┐   ┌▼──────────────────┐
    │ Discord MCP 서버 │   │ Notion MCP 서버    │
    │ (lulu-discord)  │   │ (선택 설치)         │
    │                 │   │                    │
    │ discord.js로    │   │ Notion API로       │
    │ 실제 봇 운영     │   │ 페이지 생성/편집    │
    └─────────────────┘   └────────────────────┘
```

### .mcp-lulu.json — MCP 설정 파일
```json
{
  "mcpServers": {
    "lulu": {                          // Discord MCP 서버
      "command": "node",
      "args": ["channels/lulu-discord.mjs"]
    },
    "notion": {                        // Notion MCP 서버 (선택)
      "command": "npx",
      "args": ["-y", "@notionhq/notion-mcp-server"]
    }
  }
}
```

이 설정이 Claude Code에게 "이 MCP 서버들을 시작하고, 거기서 제공하는 도구들을 사용할 수 있어"라고 알려준다.

## 11. 에이전트(Agent) — 자율적으로 일하는 AI

### 챗봇 vs 에이전트

**챗봇** (ChatGPT 등):
```
사용자: "서울 관광 코스 추천해줘"
AI: "1. 경복궁 → 2. 인사동 → 3. 남산타워..."
(끝. 사용자가 다음 질문을 해야 함)
```

**에이전트** (Lulu):
```
사용자: "내 Downloads 폴더에 있는 자격증 사진들 분석해서 스펙 목록 만들어줘"
Lulu:
  1. [Glob 도구] Downloads에서 이미지 파일 검색
  2. [Read 도구] 각 이미지 읽기 (자격증 사진 인식)
  3. 텍스트 추출 및 구조화
  4. [Write 도구] 스펙 목록 파일 생성
  5. [reply 도구] Discord로 결과 보고
  "5개 자격증 찾았음. 정리 완료: ~/Documents/spec-list.md"
```

### 에이전트의 특징
1. **자율 실행**: 중간에 물어보지 않고 여러 도구를 연속 사용
2. **판단**: 상황에 따라 어떤 도구를 쓸지 스스로 결정
3. **반복**: 필요하면 같은 작업을 반복 (예: 파일 10개를 하나씩 읽기)
4. **보고**: 완료 후 결과를 사용자에게 전달

### Lulu의 에이전트 동작 예시

Discord에서 PDF 파일을 보내며 "이거 요약해줘"라고 말하면:

```
[Discord 메시지 수신]
  ↓
[discord-channel.mjs] 파일 다운로드 → ~/Documents/Lulu_Memory/inbox/에 저장
  ↓
[MCP notification] Claude Code에게 "파일 도착" 알림
  ↓
[Claude/Lulu] Read 도구로 PDF 읽기
  ↓
[Claude/Lulu] 내용 분석 및 요약 생성
  ↓
[Claude/Lulu] reply 도구로 Discord에 요약 전송
  ↓
[Discord] 사용자에게 요약 메시지 도착
```

## 12. 하네스 엔지니어링

### 하네스(Harness)란
AI 모델을 실제 서비스로 만들기 위한 **주변 시스템 전체**를 설계하는 것.

모델 자체가 엔진이라면, 하네스는 차체, 핸들, 브레이크, 계기판이다.

### 하네스의 구성요소

```
┌─────────────────────────────────────────────────────────┐
│                    하네스 (Harness)                        │
│                                                           │
│  ┌───────────┐  ┌──────────┐  ┌───────────┐              │
│  │ 시스템      │  │ 메모리    │  │ 도구       │              │
│  │ 프롬프트    │  │ 시스템    │  │ (MCP)     │              │
│  │ (CLAUDE.md)│  │ (파일DB) │  │           │              │
│  └─────┬─────┘  └────┬─────┘  └─────┬─────┘              │
│        │             │              │                     │
│        ▼             ▼              ▼                     │
│  ┌──────────────────────────────────────────────┐        │
│  │              AI 모델 (Claude Opus)             │        │
│  └──────────────────────────────────────────────┘        │
│        │             │              │                     │
│  ┌─────▼─────┐  ┌───▼────┐  ┌─────▼──────┐              │
│  │ 세션 관리   │  │ 안전장치│  │ 입출력 채널  │              │
│  │ (hooks)    │  │ (가드) │  │ (Discord)  │              │
│  └───────────┘  └────────┘  └────────────┘              │
│                                                           │
└─────────────────────────────────────────────────────────┘
```

### 하네스 엔지니어링의 핵심 요소

| 요소 | 역할 | Lulu에서 |
|------|------|---------|
| 시스템 프롬프트 | AI의 성격과 규칙 | `CLAUDE.md` |
| 메모리 시스템 | 세션 간 정보 유지 | `.claude/memory/` + hooks |
| 도구 연결 | AI가 행동할 수 있게 | MCP 서버 (Discord, Notion) |
| 세션 관리 | 시작/종료/복구 | hooks (session-start, compact, stop) |
| 안전장치 | 위험한 행동 방지 | CLAUDE.md의 제한 규칙 |
| 입출력 채널 | 사용자와 소통 | Discord DM/채널 |
| 자동 재시작 | 중단 시 복구 | `run-lulu.sh` (while true 루프) |

### 좋은 하네스 설계의 원칙

1. **실패에 강하게**: 에러가 나도 자동 재시작 (`run-lulu.sh`)
2. **기억을 보존**: 컨텍스트 압축 전에 중요 정보 저장 (pre-compact hook)
3. **명확한 페르소나**: AI가 일관되게 행동하도록 (CLAUDE.md)
4. **점진적 확장**: MCP로 새 기능을 쉽게 추가 가능
5. **사용자 친화**: Discord처럼 익숙한 인터페이스 사용

## 13. Lulu에 적용된 모든 것

### 전체 아키텍처 (한눈에 보기)

```
사용자 (Discord)
    │
    │ 메시지 / 파일 전송
    ▼
┌──────────────────────────────────┐
│  Discord MCP 서버                 │  ← channels/discord-channel.mjs
│  (discord.js 봇이 메시지 수신)     │
│                                   │
│  1. 메시지 텍스트 추출              │
│  2. 첨부파일 다운로드               │
│  3. MCP notification 전송          │
└──────────────┬───────────────────┘
               │
               ▼
┌──────────────────────────────────┐
│  Claude Code (AI 런타임)          │
│                                   │
│  시스템 프롬프트: CLAUDE.md        │  ← "너는 Lulu, 개인 비서야"
│  메모리: .claude/memory/           │  ← "이 사용자는 경영학과 4학년"
│  세션 노트: ~/Lulu_Memory/         │  ← "어제 이력서 초안 작성했음"
│                                   │
│  AI 판단 → 도구 선택 → 실행        │
│  - Read: 파일/이미지 읽기           │
│  - Write: 결과 저장                │
│  - reply: Discord로 응답           │
│  - Notion MCP: 포트폴리오 편집      │
└──────────────┬───────────────────┘
               │
               │ reply 도구 호출
               ▼
┌──────────────────────────────────┐
│  Discord MCP 서버                 │
│  → channel.send() 실행            │
│  → Discord에 메시지 전송           │
└──────────────────────────────────┘
               │
               ▼
        사용자 (Discord)
         답변 수신!
```

### 각 파일의 역할 요약

| 파일 | 역할 | 해당 개념 |
|------|------|-----------|
| `CLAUDE.md` | Lulu의 성격, 원칙, 능력 정의 | 시스템 프롬프트, 페르소나 |
| `channels/discord-channel.mjs` | Discord ↔ AI 연결 | MCP, 도구 사용 |
| `channels/lulu-discord.mjs` | Lulu 전용 Discord 설정 | 하네스 설정 |
| `.mcp-lulu.json` | 어떤 MCP 서버를 사용할지 | MCP 구성 |
| `hooks/session-start.sh` | 세션 시작 시 메모리 로드 | 메모리, 하네스 |
| `hooks/pre-compact.sh` | 컨텍스트 압축 전 저장 | 컨텍스트 윈도우, 메모리 |
| `hooks/post-compact.sh` | 압축 후 핵심 정보 복구 | 컨텍스트 윈도우, 메모리 |
| `hooks/stop.sh` | 세션 종료 기록 | 세션 관리 |
| `run-lulu.sh` | 자동 재시작 래퍼 | 하네스 (안정성) |
| `.claude/memory/` | 장기 기억 저장소 | 메모리 시스템 |
| `SETUP.md` | 설치 가이드 | (문서) |

### 사용 시나리오 예시

**시나리오: 자격증 사진들로 포트폴리오 만들기**

```
1. Discord에서 자격증 사진 5장 전송
   "이거 분석해서 Notion에 포트폴리오 만들어줘"

2. Lulu가 자동으로:
   - 5장 다운로드 (discord-channel.mjs)
   - 각 이미지 분석 (Read 도구 — multimodal)
   - 자격증명, 발급기관, 취득일 추출
   - 메모리에 스펙 정보 저장 (장기 기억)
   - Notion에 포트폴리오 페이지 생성 (Notion MCP)
   - 완료 보고 → Discord

3. 다음 날 Discord에서:
   "어제 만든 포트폴리오에 인턴 경험도 추가해줘"
   → Lulu가 메모리에서 어제 작업 내용 로드
   → Notion 페이지 수정
```

## 용어 정리

| 용어 | 설명 |
|------|------|
| AI (Artificial Intelligence) | 인공지능. 데이터에서 패턴을 학습한 컴퓨터 시스템 |
| LLM (Large Language Model) | 대규모 언어 모델. 텍스트 생성 AI |
| 임베딩 (Embedding) | 단어/문장을 숫자 벡터로 변환하는 기술 |
| 토큰 (Token) | AI가 텍스트를 처리하는 최소 단위 |
| 컨텍스트 윈도우 | AI가 한 번에 처리할 수 있는 최대 텍스트 길이 |
| 프롬프트 (Prompt) | AI에게 보내는 지시/질문 |
| 시스템 프롬프트 | AI의 성격과 규칙을 설정하는 텍스트 |
| 페르소나 (Persona) | AI의 역할과 성격 설정 |
| 메모리 (Memory) | AI의 장기 기억을 파일로 구현한 시스템 |
| 도구 (Tool) | AI가 호출할 수 있는 외부 기능 |
| MCP (Model Context Protocol) | AI와 외부 서비스를 연결하는 표준 프로토콜 |
| 에이전트 (Agent) | 도구를 자율적으로 사용하여 작업을 수행하는 AI |
| 하네스 (Harness) | AI 모델을 실제 서비스로 만드는 주변 시스템 전체 |
| Compact (컴팩션) | 컨텍스트 윈도우 초과 시 대화를 요약/압축하는 과정 |
| Hook | 특정 이벤트(시작, 종료 등) 발생 시 자동 실행되는 스크립트 |
| Hallucination | AI가 사실이 아닌 내용을 만들어내는 현상 |
